# 2026-03-25 工作日志

## 修复 giscus 初始深色模式问题

**问题**：giscus 无论站点默认深色，都以浅色模式出现；仅手动切换后才正确。

**根因**：
1. `Layout.astro` 内联脚本 `initialColorScheme = ""`（空），`theme.ts` 为 `"dark"`，首次访问主题判断不一致
2. MutationObserver 只在 iframe 插入 DOM 时调用 `postMessage`，但 iframe 未 load 完，postMessage 无效
3. `data-theme="preferred_color_scheme"` 跟随系统（可能浅色）

**修复**：
- `Layout.astro` 的 `initialColorScheme` 统一改为 `"dark"`
- `Giscus.astro` 重构：加载前从 localStorage 读取主题存入 `window.__giscusTheme`；`data-theme` 改为 `"dark"`；监听 iframe `load` 事件后再发 postMessage；patch `window.theme.setTheme` 时加 `__giscusPatchedSetTheme` 标志防重复

## 修复 giscus 默认浅色时显示深色的问题（反向 bug）

**问题**：当站点默认为浅色时，giscus 仍然显示为深色。

**根因**：上一版 `getSiteTheme()` 中硬编码了 fallback 为 `"dark"`，当 localStorage 为空（首次访问）时错误返回深色。另外 `data-theme` 写死为 `"dark"` 也不跟随初始主题。

**修复**（2026-03-25）：
- 改为直接读取 `<html data-theme="...">` 属性作为最可靠数据源（Layout.astro 内联脚本已将正确主题写入）
- 将 giscus `client.js` 改为动态注入（`document.createElement("script")`），`data-theme` 在注入时由 `window.__giscusTheme` 动态填入，彻底消除初始主题错误
- fallback 从 `"dark"` 改为 `"light"`（更保守，与系统默认一致）
